## 1. Instalação rápida

> Funciona em Ubuntu 22.04 / 24.04. Instala PHP 8.2, Docker, drivers de banco e já sobe tudo.

Clone o repositório e rode um único comando:

```bash
git clone https://github.com/seu-repo/api_vupi.us_php.git
cd api_vupi.us_php
sudo bash install.sh
```

Isso faz tudo automaticamente:
- Instala PHP 8.2 + extensões pdo_pgsql e pdo_mysql
- Instala Docker + Docker Compose
- Instala Composer
- Configura firewall (UFW)
- Instala dependências do projeto (`composer install`)
- Cria o `.env` a partir do `EXEMPLO.env`
- Gera `JWT_SECRET` e `JWT_API_SECRET`
- Sobe o banco via `docker-compose.yml`
- Roda migrations e seeders
- Inicia o servidor PHP

Ao final o servidor estará rodando em `http://0.0.0.0:3005`.

## 2. Instalação manual

Se preferir controlar cada etapa, use o menu interativo:

```bash
# 1. Clone e instale dependências
git clone https://github.com/seu-repo/api_vupi.us_php.git
cd api_vupi.us_php
composer install

# 2. Menu interativo de setup
php vupi setup
```

O menu oferece:

```
1)  Preparar .env
2)  Subir banco via docker-compose (recomendado)
3)  Criar banco via docker run (container avulso)
4)  Rodar migrations
5)  Rodar seeders
6)  Subir servidor PHP
7)  Subir servidor com PM2
8)  Validar conexão com banco
9)  Executar tudo automaticamente
10) Gerar JWT_SECRET e JWT_API_SECRET
11) Gerar token JWT de API
0)  Sair
```

Ou rode tudo de uma vez sem interação:

```bash
php vupi setup --auto
```

Flags disponíveis:

| Flag | Valores | Padrão | Descrição |
|------|---------|--------|-----------|
| `--db-mode` | `compose`, `docker`, `skip` | `compose` | Como criar o banco |
| `--server` | `php`, `pm2` | `php` | Como iniciar o servidor |
| `--jwt` | `if-empty`, `skip` | `if-empty` | Gerar secrets JWT |
| `--api-token` | `generate`, `skip` | `skip` | Gerar token de API |

## 3. Configuração do .env

O arquivo `.env` é criado automaticamente a partir do `EXEMPLO.env`. Edite conforme seu ambiente:

```ini
# Aplicação
APP_ENV=production        # development | production
APP_DEBUG=false           # true só em desenvolvimento
APP_PORT=3005
APP_URL=https://seudominio.com

# Banco de dados
DB_CONEXAO=postgresql     # postgresql | mysql
DB_HOST=localhost
DB_PORT=5432
DB_NOME=vupi_db
DB_USUARIO=admin
DB_SENHA=sua_senha_segura

# JWT (gerados automaticamente pelo setup se vazios)
JWT_SECRET=
JWT_API_SECRET=
JWT_ISSUER=https://seudominio.com
JWT_AUDIENCE=https://seudominio.com
JWT_EXPIRATION_TIME=3600

# Cookies
COOKIE_SECURE=true        # true em produção (HTTPS)
COOKIE_SAMESITE=Lax
COOKIE_DOMAIN=seudominio.com   # sem https://
```

> Nunca coloque `https://` no `COOKIE_DOMAIN`. Use apenas o domínio: `seudominio.com`

## 4. Comandos do CLI

### Setup

```bash
php vupi setup                    # menu interativo
php vupi setup --auto             # tudo automático
php vupi setup --help             # ajuda
```

### Migrations e seeders

```bash
php vupi migrate                  # roda migrations
php vupi migrate --seed           # migrations + seeders
php vupi migrate --rollback       # desfaz última migration
```

### Módulos e plugins

```bash
php vupi make:module NomeModulo   # cria estrutura de módulo
php vupi make:plugin NomePlugin   # cria estrutura de plugin
php vupi plugin:migrate           # migrations de plugins
php vupi plugin:rollback          # rollback de plugin
php vupi plugin:validate          # valida plugins instalados
php vupi capability:list          # lista capabilities disponíveis
```

### Servidor

```bash
# Desenvolvimento
php -S 0.0.0.0:3005 index.php

# Produção com PM2
pm2 start php --name vupi.us-api -- -S 0.0.0.0:3005 index.php
pm2 save
pm2 logs vupi.us-api
```

### Make (atalhos)

```bash
make setup        # sobe banco + migrations + seeders
make migrate      # só migrations
make seed         # migrations + seeders
make up-pg        # sobe PostgreSQL + Adminer
make up-mysql     # sobe MySQL + Adminer
make down         # para containers
make clean        # limpa cache
make test         # roda testes de segurança
make install-env  # instala tudo no Ubuntu (equivale a sudo bash install.sh)
```

## 5. Arquitetura

```
api_vupi.us_php/
├── src/
│   ├── Kernel/          # núcleo: container, router, middlewares, DB
│   └── Modules/         # seus módulos ficam aqui
├── public/              # arquivos estáticos
├── index.php            # entry point HTTP
├── vupi.us              # entry point CLI
├── install.sh           # instalação completa em um comando
└── docker-compose.yml   # PostgreSQL + MySQL + Adminer
```

O kernel nunca é modificado. Toda a lógica da aplicação vive em `src/Modules/`.

## 7. Rotas e middlewares

Edite `src/Modules/Financeiro/Routes/web.php`:

```php
<?php
use Src\Modules\Financeiro\Controllers\FaturaController;
use Src\Kernel\Middlewares\AuthHybridMiddleware;
use Src\Kernel\Middlewares\AdminOnlyMiddleware;

// Rota pública
$router->get('/api/faturas', [FaturaController::class, 'listar']);

// Rota autenticada (JWT via cookie ou Bearer)
$router->post('/api/faturas', [FaturaController::class, 'criar'], [
    AuthHybridMiddleware::class,
]);

// Rota restrita a admin_system
$router->delete('/api/faturas/{id}', [FaturaController::class, 'deletar'], [
    AuthHybridMiddleware::class,
    AdminOnlyMiddleware::class,
]);
```

Middlewares disponíveis:

| Middleware | Descrição |
|-----------|-----------|
| `AuthHybridMiddleware` | JWT via cookie ou Bearer — retorna JSON 401 |
| `AuthPageMiddleware` | JWT via cookie — redireciona para `/` se não autenticado |
| `AdminOnlyMiddleware` | Exige `nivel_acesso = admin_system` |
| `ApiTokenMiddleware` | Valida JWT assinado com `JWT_API_SECRET` |
| `RateLimitMiddleware` | Limite de requisições por IP |

## 8. Injeção de dependência

O container resolve dependências automaticamente pelo construtor:

```php
namespace Src\Modules\Financeiro\Controllers;

use Src\Modules\Financeiro\Services\FaturaService;
use Src\Kernel\Http\Response\Response;

class FaturaController
{
    public function __construct(private FaturaService $service) {}

    public function listar(): Response
    {
        return Response::json($this->service->todas());
    }
}
```

Nenhum registro manual necessário.

## 9. Comunicação entre módulos

### Dependência obrigatória

```php
use Src\Modules\Usuario\Services\UsuarioService;

class FaturaService
{
    public function __construct(private UsuarioService $usuarios) {}
}
```

Se o módulo `Usuario` não existir, o sistema lança erro no boot — comportamento correto.

### Dependência opcional

Use `?` (nullable) para dependências que podem não existir:

```php
use Src\Kernel\Contracts\EmailSenderInterface;

class FaturaService
{
    public function __construct(
        private ?EmailSenderInterface $email = null
    ) {}

    public function emitir(): void
    {
        $this->salvarNoBanco();
        // Só envia e-mail se o módulo de e-mail estiver instalado
        $this->email?->sendCustom('cliente@email.com', 'Fatura', '<p>Gerada!</p>');
    }
}
```

## 10. Segurança

### Checklist de produção

- `APP_DEBUG=false` — nunca exponha stack traces
- `APP_ENV=production`
- `COOKIE_SECURE=true` — obrigatório com HTTPS
- `COOKIE_DOMAIN=seudominio.com` — sem `https://`
- `JWT_SECRET` e `JWT_API_SECRET` com valores aleatórios fortes (gerados pelo setup)
- Banco com senha forte (não use `123456`)
- Cloudflare ou proxy reverso na frente do servidor

### O que já está protegido

- Senhas com Argon2id
- Tokens JWT com blacklist (logout invalida o token)
- Rate limiting por IP em todas as rotas de auth
- Audit log de eventos de segurança (`audit_logs`)
- Detecção de brute force (alerta após 10 falhas em 5 min)
- Timing attack mitigation no login (200ms mínimo)
- Cookies com `HttpOnly`, `Secure`, `SameSite`
- Headers de segurança em todas as respostas
- Prepared statements em todas as queries (sem SQL injection)
- `expose_php = Off` e `X-Powered-By` removido

### Endpoints de autenticação

```
POST /api/auth/login          # login, seta cookie HttpOnly
POST /api/auth/logout         # invalida token e remove cookie
GET  /api/auth/me             # retorna usuário autenticado
POST /api/auth/refresh        # renova access token
POST /api/auth/register       # cadastro de usuário
POST /api/auth/forgot-password
POST /api/auth/reset-password
```
